## 5. Critical Path Layout Discrepancy

The upstream archive uses:
```
PaddleOCR/Det|Rec/V5/PP-OCRv5_mobile_rec_infer/slim.onnx
PaddleOCR/Det|Rec/V5/PP-OCRv5_mobile_rec_infer/inference.yml
```

The B11.4 manifest expects:
```
PaddleOCR/Det|Rec/V5/ppocr_det_v5.onnx
PaddleOCR/Det|Rec/V5/inference.yml
```

This affects all 10 PaddleOCR ONNX entries and all 7 inference.yml entries. The ONNX filenames (`slim.onnx` vs `ppocr_*.onnx`) and model subdirectory names (`PP-OCRv5_mobile_rec_infer/`) differ.

Additionally, V4En shares `Rec/V4/` (not a separate `Rec/V4En/` directory), and Latin/Eslav/Korean share `Rec/V5/` (not separate directories per language variant).

**Implication**: B11.4 registry paths and the B11.5 manifest must be aligned with the actual upstream layout. This is a B11.4 correction.

## 7. inference.yml Compatibility

**All 7 Rec inference.yml files contain inline `PostProcess.character_dict` as a YAML sequence.** This confirms they are compatible with the current `ParseInferenceYml()` parser.

Notable observation: Det V4/V5/V6 directories also contain `inference.yml` files (detection config), but these are not used by the recognition parser and contain no `PostProcess.character_dict`.

Rec CN (Chinese) inference.yml files are large (~166 KB, ~18,000 lines) due to their extensive character dictionaries. Language-specific variants (English, Latin, Eslav, Korean) have smaller dictionaries (1.5–108 KB).
